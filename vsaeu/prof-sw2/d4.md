# Blog Plattform mit MVC Pattern

1. Model
    Artikel und Kommentare speichern:
    Es braucht jeweils Tabellen mit folgenden Attributen
    Artikel: id, titel, inhalt, Erstelldatum, Änderungsdatum, Verfasser-id

    Kommentar: id, content, erstelldatum, Ändeurngsdatum, Verfasser-id, artikel-id

    User: 
    Man muss user registrieren und eine Felder müssen updatefähig sein. Der User kann seinen Account auch löschen lassen:
    Attribute: id, Profilname, email, Geburtsdatum, agbs_akzeptiert

    Beziehungen:

    Ein User kann viele Artikel und Kommentare verfassen.
    Ein Artikel kann viele Kommentare haben.
    Ein Kommentar gehört zu einem Artikel und einem User.

2. View

    # Artikel: Titel, Content mit Bildern, Verfasser, Kommentare

    # Artikel-erstellung: 
    Input für Titel
    Beliebig viele Passagen über Textareas einfügbar
    Möglichkeit vor oder nach jeder Passage ein Bild mit einer Beschreibung einzufügen (Upload Button + preview)
    Vorschau des fertigen Artikels

    # Kommentare in chronischer Reihenfolge und Pagiation oder virtual scroll
    Neue Kommentare über ein Input/Textarea unterhalb der bisherigen Kommentare.
    Möglichkeit auf vorherige Kommentare zu antworten und diese zu zitieren.

3. Controller
Ein Eintrag wird durchgeführt. Wird an Controller geschickt. Dieser verarbeitet/validiert den Eintrag und schickt diesen an die Datenbank. Von dort werden die gespeicherten Daten sogleich wieder geholt und die View wird aktualisiert.
