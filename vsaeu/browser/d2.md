1: GET
- 9 Headers
- Response entspricht dem Verzeichnis von dem aus der http-Server gestartet wurde.
- Klicke ich auf eine html Datei wird diese in der Vorschau angezeigt

--> Sinnvoll wenn man etwas aufrufen möchte

2: POST
- 5 Headers
- Kein Body
- 405 Method Not Allowed

--> Beim eintragen einer neuen Ressorce

3: PUT
- Programm hängt sich auf

--> Beim aktualisieren von Teilen einer Ressource, z.B. Spalten in einer Datenbank-Zeile

4: Delte
- 405 Method Not Allowed
- 5 Headers
- kein Body

--> Löschen von zeilen in einer Datenbank

5: Options: 
- identisch zu POST und DELTE

--> Fragt Server welche Methoden erlaubt sind
